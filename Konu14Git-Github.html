Git ve Github Kullanımı

Git : versiyon kontrol sistemidir.

Github : git sistemini kullanarak projelerimizi yönetip depolayan uygulamadır.

Önemli Git Terimleri

Repository Kısaca repo, dosyalarımızın bulunduğu projeler
Master Projemizin kararlı sürümü, ana gövdesi
Branch Git, ağaç mantığı ile çalışır ve proje birden fazla dala ayrılıp bu şekilde geliştirilebilir. Sonrasında bu dallar yine master da birleştirilip proje tamamlanır.
Merge Branch'lerdeki değişikliklerin master da birleştirilme işlemidir
Push Bilgisayardaki kodlarımızı github daki repoya(projenin kendi deposuna) göndermemizi sağlar.
Pull Github daki projemizin son halini bilgisayarımıza çekmemizi sağlar
.gitignore Git in takip etmemesini istediğimiz dosyaları (lokalde denediğimiz resimler vb) push işleminde repoya gönderilmesini engellememizi sağlar.
Untracked(U:izlenmeyen) Git tarafından henüz izlenmeyen yeni eklenmiş dosyalar
Unstaged(Hazırlanmamış) Güncellenmiş fakat commitlenmek için hazırlanmamış dosyalar.
Staged(Hazırlanmış) Commitlenmeye hazır olan dosyaları ifade eder
Deleted(Silinmiş) Projeden silinmiş fakat Git üzerinden kaldırılmamış dosyaları ifade eder.

Gitignore

Git in takip etmemesini istediğimiz dosyaları (lokalde denediğimiz resimler vb) push işleminde repoya gönderilmesini engellememizi sağlar.

Gitignore dosyası oluşturmak için sol taraftaki proje dosyalarımızın olduğu kısımdaki proje adının bulunduğu yerden + ikonuyla yeni doya ikonuna tıklayıp bir isim yazmadan sadece .gitignore yazıp enter ile dosyayı oluşturuyoruz.
